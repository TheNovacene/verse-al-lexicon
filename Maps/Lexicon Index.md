## Live Queries

These queries auto-update as you add, edit, or retag entries. They render in **Reading view** or **Live Preview** вАФ in raw edit mode they show as code blocks. Edit any of them to fit a new question.

### All entries (master table)

```dataview
TABLE without id
  link(file.name, title) as "Term",
  glyph-prefix as "Glyph",
  glyph-family as "Family",
  version as "Introduced",
  status as "Status"
FROM "Entries"
SORT file.name ASC
```

### Entries grouped by introduction-version

```dataview
TABLE without id
  link(file.name, title) as "Term",
  glyph-family as "Family"
FROM "Entries"
SORT version ASC, file.name ASC
GROUP BY version
```

### The v7 polished set

```dataview
TABLE without id
  link(file.name, title) as "Entry",
  glyph-family as "Family",
  domains as "Domains"
FROM "Entries"
WHERE version = "v7"
SORT file.name ASC
```

### The v8 Verse-Nerves cluster

```dataview
TABLE without id
  link(file.name, title) as "Entry",
  glyph-family as "Family",
  domains as "Domains"
FROM "Entries"
WHERE version = "v8" OR contains(file.tags, "#revised/v8")

```

### Counts per version

```dataview
TABLE without id
  version as "Version",
  length(rows) as "Count"
FROM "Entries"
GROUP BY version
SORT version ASC
```

### Distribution by glyph-family

```dataview
TABLE without id
  glyph-family as "Family",
  length(rows) as "Count"
FROM "Entries"
GROUP BY glyph-family
SORT length(rows) DESC
```

### Recently edited

```dataview
TABLE without id
  link(file.name, title) as "Entry",
  file.mtime as "Last edited"
FROM "Entries"
SORT file.mtime DESC
LIMIT 10
```
